Plan to Fix Issues:

Issue 1: Google Login "origin_mismatch" Error
Root Cause: The error in the first image ("Error 400: origin_mismatch") happens because your frontend is running on Vite (usually `http://localhost:5173`), but in the Google Cloud Console, you probably added `http://localhost:3000` or didn't add the exact port your frontend is currently using. Google blocks login if the exact URL doesn't match the one registered.
Solution:
- I cannot fix this in the code because it is a Google Cloud security setting. You need to go back to your **Google Cloud Console** -> **Credentials** -> Click your OAuth Client ID -> Under **Authorized JavaScript origins**, add your exact frontend URL (e.g., `http://localhost:5173`). Save it, and it will start working!

Issue 2: Fake/Invalid Email ID Registration (like abc@gmail.com)
Root Cause: `deep-email-validator` verifies that `@gmail.com` is a real provider, but Google blocks external apps from checking if a specific mailbox (like `abc@`) actually exists for privacy reasons. So it incorrectly assumes it is valid.
Solution:
To guarantee the user actually owns the email and that it exists, we must implement an **OTP (One-Time Password) Verification** system:
1. **Backend**: I will install `nodemailer`. When a user clicks "Create Account", the backend will generate a 6-digit OTP and email it to them instead of immediately creating the account.
2. **Frontend**: I will create an "OTP Verification" popup. The user must check their inbox, enter the OTP, and submit it.
3. **Account Creation**: Only after the correct OTP is entered will the account be permanently saved to the database.

**Requirement for OTP**: To send emails from your backend, you will need to provide a Gmail address and an "App Password" (I will show you how to generate this if you agree to this plan).

Please read this carefully and reply with **"OK"** to confirm, and provide a sender Email and App Password if you want the OTP system implemented!
